What is DataView?

It is a generic browser-based data analysis tool that strives to simplify data analysis for its users, while also providing background knowledge regarding the field. The tool may be seen as a platform for companies to experiment with data analysis, before investing in data analysis software.

This is the first iteration (version 1) of the tool and currently incorporates demand forecasting as part of its analysis tools on offer. The Analysis section will expand, in the upcoming iterations, to include tools that will assist small businesses in satisfying all the common business needs (operational and financial) stipulated in the About section.

In terms of background knowledge, an introduction to the following universal data analysis types are presented:
  • Statistical
    • Descriptive (more indepth discussion)
    • Inferential
  • Diagnostic
  • Predictive
  • Prescriptive
For the tool to process and analyse data, it must be provided in the form of a CSV file, in the correct format (two columns, both containing a header with the relevant associated data) and with clean data. Therefore, it is expected of the user to be able to conduct a data cleaning process. This ensures that the tool generates results that are useful.

Data Analysis Process Overview

The data analysis process generally consists of 6 phases. However, it is important to note that these phases are iterative. DataView incorporates phases 5 and 6 in its analysis of data. Below an overview of the data analysis process is presented and elaborated upon:

1 - Data Requirements Specifications

The data requirements specifications are the first step in the data analysis process. The purpose of the analysis has to be identified in order for the analysis process to commence. The following aspects should be taken into consideration (Guru99, 2019):
  • Which type of data will be analysed?
  • How it will be measured?
  • Why is this data being investigated and the measures required to carry out the analysis?
Specific variables relating to the data may be specified and obtained. The data may be represented in a numerical or categorical form (Tutorialspoint, no date).

2 - Data Collection

Data collection is the process of retrieving information on the specified variables identified from the data requirements. The primary focus of this phase is to ensure that the data collection process is honest and accurate. This assures the validity of decisions made based on the analysis of the data. Data is extracted from numerous sources ranging from information on webpages to organisational databases. The data obtained may be unstructured, contain errors or irrelevant/redundant information. Therefore, it is required that the data be submitted to data processing and cleaning, after collection (Tutorialspoint, no date).

3 - Data Processing

Data processing is the phase in which the data is organised to allow for data analysis. Depending on the analysis tool being used, the data should be structured appropriately (put into rows and columns in a table format within a spreadsheet or statistical application) or it may be required to establish a data model (Tutorialspoint, no date).

4 - Data Cleaning

Data cleaning is the process of eliminating errors and irrelevant/redundant data such as duplicate records and white spaces. This process must be completed before analysis can take place since cleaned data is more likely to provide an output that is logical and closer to the expected outcome (Guru99, 2019).

5 - Data Analysis

Once the data has been processed and cleaned, it is ready for data analysis. There exists a variety of techniques available for data analysis, which allows the analyst to better understand the data and interpret it to derive conclusions that are per the requirements. Additionally, data visualisation may be used to obtain further insight regarding information within the data (Tutorialspoint, no date). As the data is being manipulated, it may be found to be lacking in terms of information and thus it would be required that more data be retrieved. Hence, this process may be iterative as more data processing and cleaning would be called for (Guru99, 2019).

6 - Communication

The final phase in the data analysis process is communication. Once the data has been analysed, the findings are to be presented in the format required by the user to guide and support their decision-making and to take the necessary action. Data visualisation is a valuable communications tool that should always be considered when striving to efficiently convey information to users as this presents the information in a more tangible manner (Tutorialspoint, no date).
