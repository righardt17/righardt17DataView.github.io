Help

General

There are several means of navigation, with the primary being the top navigation bar and the side slide-in navigation bar under for the analysis section.

Top navigation bar

The top navigation bar allows you to navigate through the main sections of the tool. The "Top" button (furthest right) is used to return to the top of the page.

Side slide-in navigation bar

The side slide-in navigation is accessed by clicking on "Analysis". Three sections are presented, as shown above, with each having their own subsections. The "Introduction" section provides information on the various general analysis types. The "Operational" section will focus on analysis that is closely associated with sales, inventory, marketing, and workforce, while the "Financial" section will focus on analysis closely associated with cashflow and investment risk.

Introduction section navigation bar

The introduction section navigation bar allows for quick transitioning between the various analysis types being discussed.

Open the Excel document containing your data.

Select "File" in the left corner of the task bar.

Select the "Save As" option listed.

Click on the field below the field containing your file name (in this case 'Data') and select "CSV UTF-8(Comma delimted) (*.csv)". The file will now be saved as the required CSV file.

The pages are designed to adjust according to the size of the available window. The best viewing experience is full screen, but the analysis pages were designed with multi-tasking in mind. Therefore, it is advised to use the analysis tools in half-screen windows for better screen-output value or full screen for larger visuals.

If you want to report a bug, ask questions or provide feedback regarding the tool please fill out the form presented under the "Contact" section.

Otherwise, if you want to contact the author please make use of the channels provided. The author is bilingual (Afrikaans and English) so feel free to make contact in whichever language you prefer.

Analysis

Demand Forecast

The demand forecasting tool requires a specific data format (apart from the file being a CSV file) in order for it to be analysed. Below is an example of the required format:

As seen from the above example, the columns containing the periods and associated demand must have a header (in this example: "Period" and "Demand").

The period data, may be presented in another format that provides the best context for yourself, for example:
  • "Q1 2013"
  • "2007"
  • "March"
  • "Day 1" etc.

The demand data should be presented in the form of unit sales as this is the best indication of actual growth or decline of customer demand.

Note - the first column must contain the period data and consequently the demand data in the second column.

Other input data (not demand data) may be used for analysis, but at your own discretion. The implemented forecast models will provide information regarding underlying patterns in the data. However, these models do not take other external factors into account - bear this in mind when using the results.

In the case where the graphs being displayed change datasets when hovering over them, reload the page or press the "Clear" button. This occurs when the previous dataset was not cleared before selecting a new one for analysis.

The demand forecasting section includes a brief introduction to the forecasting models that are utilised. This is followed by an input section where the user is requested to provide the CSV file to be used for the demand forecasting analysis.

The input section allows the user to select the desired CSV file by clicking on the "Choose File" button. Whilst the Clear button allows the user to clear the file selection and provide a new dataset for analysis.

After the desired CSV file has been provided, the analysis process starts, and the forecasted results are provided under the Results Summary section. This section consists of the forecast demand results, the prediction confidence intervals ranging from 50% to 99% as well as the forecast error measures for each model.

The Download Results Summary as PDF button allows the user to download a pdf containing a summary of the forecast results.

The error measure table provides the user with important statistical information that allows for a quick comparison regarding the forecast error measures of each model to help establish which model is the best fit for the dataset being analysed.
  • MSE - Mean Squared Error
  • MAD - Mean Absolute Deviation
  • MAPE - Mean Absolute Percentage Error
  • TS Range - Test Signal Range (refers to the bias range, where TS < 0 and TS > 0 is under and over forcasting respectively)
  • Average TS - On average does the model under or over forecast
  • Feasibility - Provides an indication (based on the MAPE) regarding which model has the best fit for the dataset

The confidence interval tables allow the user to view the estimate demand range at certain confidence intervals assuming that the error distribution of each forecast model is somewhat normal (kurtosis value is relatively close to zero or within the -2 to +2 range) and that the forecast models are not biased (TS range is within ±6).

After the Results Summary section, each model is provided with a graphical presentation of its fit on the input dataset with its predicted future demand as well as the 95% confidence interval of said predicted demand. Each model’s error distribution, forecast error measures and error monitoring charts are also displayed to the user.

The figure above shows a graphical illustration of the input dataset (previous sales demand) in the form of a bar graph, Winter’s model’s forecast results (based on the input data) as well as the 95% confidence interval of the predicted demand in the form of a line chart. The labels of the y-axis, x-axis and the forecast model-specific parameters (α, β and γ) are provided above the graph canvas. The legend for each dataset plot is provided with their associated colour. The graphs are interactive – the user is therefore provided with the option to hide and unhide certain plots by simply clicking on the legend of the desired dataset. Information regarding specific data points, such as the actual- or forecasted demand for a period, is made available to the user in the form of a popup box by hovering over the bars of the bar graph or data points of the line plot.

The frequency graph provides the user with the error distribution of Winter’s model – indicating the range of error values between the actual demand and the predicted demand as well as the frequency of those error values. This also allows the user to visualise the bias of the forecast model (how often the model over- or under- forecasts and by how much). The shape of the graph (a bell-curved shape is desired) along with the kurtosis value and the TS range provides the user with an indication regarding the acceptability of assuming that the error distribution is normal. Therefore, aiding in the validation regarding the use of the prediction confidence intervals of each forecast model.

The Control Chart (3σ) enables the user to establish whether the error of the forecast model is within normal terms (inside 3σ of the desired error of zero).

The TS Chart helps to identify whether the model is biased (outside the range of ±6) thus over-forecasting where TS > 6 and under-forecasting where TS <-6.
